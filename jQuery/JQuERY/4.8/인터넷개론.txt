4.http

1)client의 요구 - sever의 응답 (cliest-sever는 모두 객체)
2)stateless 상태 - 해당하는 stage에만 연결하는 방식으로 상시 모든 연결상태를 유지하지 않는 것을 말함
cs에서 서버가 클라이언트의 상태를 보존하지 않음을 의미
3)header: 통신의 기본정보를 담는 부분 - body: 실제로 전달하고자 하는 메시지를 담는 부분

ㄴ메시지헤더: http 메시지의 부가적인 정보를 포함(날짜, 버전정보, 쿠키와 캐시에 대한 정보)
일반헤더: 요청과 응답에 적용되지만 전송 데이터와는 관련이 없는 헤더 - 요청, 응답
요구헤더: 클라이언트 자체에 대한 자세한 정보를 포함하는 헤더 - 요청
응답헤더: 위치 또는 서버 자체에 대한 부가적인 정보를 갖는 헤더 - 응답
엔티티헤더: 메시지 바디의 컨텐츠를 나타내는 헤더  - 요청, 응답

ex) Content-type (header) : text/html

6. http 수행 방식 = method
https://youwjune.tistory.com/42
서버에서 요청과 응답이 이뤄지는 방식

-get
: 요청의 타입이 가장 단순하다
: 정적인 자원(http)에 접근하기 위한 요청에 자주 사용 된다
: 길이에 제한이 있다
: 파라미터를 사용함으로써 정보를 검색하는데 사용 => 조회
* 정적 자원: 웹 브라우저가 항상 같은 응답 데이터를 받아서 url에 해당하는 자원을 출력

-post
:데이터를 메시지 바디에 포함(url에 사용자 데이터가 없음), 입출력 스트림(io stream)을 통해 처리
:데이터의 크기에 제한이 없다.
:복잡한 정보를 전송할 필요가 있을 때 사용
:파라미터의 요청이 바디 내로 전송 된다.
:mutl-part = 여러 가지 타입의 데이터를 한꺼번에 전송 가능 - 다양한 데이터? => 전송
ㄴ io stream:
* 동적 자원: 조건에 따른 응답 데이터를 url에 전송하는 자원을 출력
-head: get과 동일하지만, 헤더 정보를 조회하고, 응답을 반환하지 않음
-put: 서버의 리소스를 대체, 혹은 생성
-delete: 서버의 리소스를 삭제
-patch: 서버의 리소스 일부를 수정
-options: 서버가 지원하는 http 메서드를 반환

-XML: 사용자 지정 html, 태그 사용 , 정확한 검색 가능 => XSL
